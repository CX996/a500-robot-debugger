# A-500机器人调试器界面部署说明

## 项目概述
本项目是一个为A-500智能焊接协作机器人设计的响应式Web调试界面，采用纯静态HTML、CSS和JavaScript开发，无需后端服务器支持。

## 部署准备

### 所需文件
项目包含以下核心文件：
- `index.html` - 主页面文件
- `style.css` - 样式表文件
- `script.js` - 交互脚本文件
- `welding-scene.svg` - 焊接场景图像

### 部署环境要求
- 任何支持静态文件服务的Web服务器（如Apache、Nginx、IIS等）
- 或云托管服务（如GitHub Pages、Netlify、Vercel等）
- 或本地开发服务器（用于测试）

## 部署方法

### 方法1：本地测试部署

#### Windows系统
1. 打开命令提示符（CMD）
2. 导航到项目目录：
   ```
   cd "c:\Users\xxxx\Documents\BaiduSyncdisk\01_XiongGu\06_焊接场景搭建\03_软件调试界面设计"
   ```
3. 使用Python启动本地服务器（需要安装Python）：
   ```
   python -m http.server 8000
   ```
4. 在浏览器中访问：`http://localhost:8000`

#### Mac/Linux系统
1. 打开终端
2. 导航到项目目录：
   ```
   cd /path/to/project
   ```
3. 使用Python启动本地服务器：
   ```
   python3 -m http.server 8000
   ```
4. 在浏览器中访问：`http://localhost:8000`

### 方法2：传统Web服务器部署

1. 将所有项目文件上传到Web服务器的网站根目录
2. 确保Web服务器配置为允许访问静态文件
3. 通过服务器域名或IP地址访问网站

### 方法3：云托管服务部署

#### GitHub Pages
1. 将项目上传到GitHub仓库
2. 在仓库设置中启用GitHub Pages
3. 选择分支（通常为main或master）和根目录
4. 等待自动部署完成
5. 通过分配的GitHub Pages域名访问网站

#### Netlify/Vercel
1. 访问Netlify或Vercel网站
2. 使用GitHub/GitLab/Bitbucket账号登录
3. 选择要部署的仓库
4. 按照提示完成部署配置
5. 等待部署完成，获得访问URL

## 维护说明

### 更新内容
1. 直接修改相应的HTML、CSS或JavaScript文件
2. 重新上传到服务器或触发云服务重新部署

### 自定义配置
- 可以通过修改CSS变量调整主题颜色
- 可以在JavaScript中修改数据更新频率
- 可以替换SVG图像为实际的机器人场景

## 故障排除

### 页面无法加载
- 检查文件路径是否正确
- 确保Web服务器已正确配置
- 检查浏览器控制台是否有错误信息

### 样式显示异常
- 确保CSS文件路径正确
- 检查浏览器是否支持CSS Grid和Flexbox
- 清除浏览器缓存

### 交互功能不工作
- 确保JavaScript文件路径正确
- 检查浏览器控制台是否有JavaScript错误
- 确保浏览器支持ES6语法
